shell编程：
一.变量：变量类型默认是字符串，这个一定要注意
     1.变量的设定：
  • 注意“”和‘’在变量内的不同：“”内特殊字符表示原有含义，‘’仅表示一般纯文本字符
  • 变量名称不能以数字开头
  • 为变量扩增内容：var=“$var”xxx
  • 若要让该变量在其他子程序中执行，要用export使其变成环境变量
     2.删除：
  • ${变量#关键词}：从前往右删除最短的；
  • ${变量##关键词}：从前往右删除最长的
  • ${变量%关键词}：从后往前删除最短的；
  • ${变量%%关键词}：从后往前删除最长的
     3.替换：
  • ${var/原来的子串/要替换的部分}：只替换符合的第一个字符串
  • ${var//原子符川/要替换的内容}：替换所有符合条件的字符串

二.正则表达式：
  • 需要注意语系环境
     1.特殊字符：如果想将下面特殊字符作为普通字符，就需要加上转义符“\”
  • ^：指向一行的开头，注意让反向选择^的区别
  • $：指向一行的结尾
  •  .：任意单个字符
  • [ ]：括号内是一个列表或范围，但只能是这个列表或范围内的某一个；注：^在括号内是反向选择，在括号外表示第一个字符；反向选择好像只有这一个用法？
     2.特殊匹配模式：
  • [:alnum:]：任意字母与数字字符
  • [:alpha:]：字母（a-zA-Z）
  • [:ascii:]：ASCII字符
  • [:blank:]：空格或制表符
  • [:space:]：空白字符，包括垂直制表符
  • [:digit:]：任意数字（0-9）
  • [:lower:]：代表小写字符，即a-z
     3.匹配字符
  • ？：匹配可选但最多匹配一次，即0个或一个
  • *：匹配0次或多次
  • +：匹配一次或多次
  • {n}：必须匹配n次，使用大括号时两边均须加上转义字符“\”
  • {n，}：必须匹配n或n次以上
  • {n，m}：匹配次数在n到m次，包括n和m
shell脚本的执行：
     sh：在子进程中执行
     source：子父进程中执行
shell script默认变量
  1. $0,$1,$2, ... 依次代表脚本名称，第一个参数，第二个参数
  2. $#：代表脚本后接的参数个数
  3. $@：代表$1,$2,$3等之意，会列出所有命令行参数
  4. $*：同上效果，默认分隔符为空格
三.数据重定向
     重定向符：
  • 标准输入：代码为0，<，<<
  • 标准输出：代码为1，>，>>
  • 标准错误输出：代码为2，2>，2>>
  1. 正确和错误输出到同一个文件：>file 2>&1或 &>file 

在shell编程中，遇大段文本或代码时，经常会用到eof。本文就介绍下其用法，供初学的朋友参考。
在shell脚本中，通常将EOF与 << 结合使用，表示后续的输入作为子命令或子Shell的输入，直到遇到EOF为止，再返回到主Shell。
EOF只是一个分界符，当然也可以用abcde替换。

shell脚本调试：
调试脚本的选项如下所示：
  • -n：读一遍脚本中的命令但不执行，用于检查脚本中的语法错误
  • -v：一边执行脚本，一边将执行过的脚本命令打印到标准错误输出
  • -x：提供跟踪执行信息，将执行的每一条命令和结果依次打印出来
使用这些选项有三种方法：
  • 在命令行提供参数
  • 在脚本开头提供参数：sh -x ./script.sh
  • 在脚本中用set命令启用或禁用参数
